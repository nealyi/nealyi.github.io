GitHub Pages + Hexo(主题：Yilia) 搭建个人博客

2016-12-01

 

  我不是很喜欢写博客，不是因为懒，而是不知道写些什么东西。因为关于技术文章，网上有很多，我能找到我想知道问题的答案，我了解的知识网上也有答案。但是对于某些问题可能很少人才会产生疑惑，所以网上的答案相对来说较少。而身为IT行业的一员，开源和共享融入我的理念，所以我想把这些东西记录下来，或许能帮助到和我有同样问题的人，同时也记录下我的生活。于是今天我就决定搭建我的个人博客。

  首先感谢令狐葱写的这篇文章，使我能顺利的搭建起这个blog。但是我为什么还要重复再写一篇，是因为我在搭建的过程中思绪还是有些乱，所以我想写一篇文章记录下我整个搭建的过程。

一. Create a new repository

  首先在Github的Create a new repository的页面创建一个新的repository。名字命名格式为
username.github.io ,username就是你在GitHub的用户名（或者组织名称）。

  Install Hexo

  当所有必要软件都安装了，你就可以使用npm来安装Hexo.

这里有必要提下Hexo常用的几个命令：

  1. hexo generate (hexo g) 生成静态文件，会在当前目录下生成一个新的叫做public的文件夹
  2. hexo server (hexo s) 启动本地web服务，用于博客的预览
  3. hexo deploy (hexo d) 部署播客到远端（比如github, heroku等平台）

三. 部署Hexo到Github Pages

   这一步恐怕是最关键的一步了，让我们把在本地web环境下预览到的博客部署到github上，然后就可以直接通过https://jiji262.github.io访问了。

   首先需要明白所谓部署到github的原理。

  1. 之前步骤中在Github上创建的那个特别的repo（username.github.io）一个最大的特点就是其master中的html静态文件，可以通过链接https://username.github.io来直接访问。
  2. Hexo g 会生成一个静态网站（第一次会生成一个public目录），这个静态文件可以直接访问。
  3. 需要将hexo生成的静态网站，提交(git commit)到github上。

明白了原理，怎么做自然就清晰了。

 使用hexo deploy部署：

   修改根目录下的_config.yml，修改deploy如下,将username改为自己的Github的名称，就是第一步我们新建的repository：

deploy:
    type: git
    repo: git@github.com:username/username.github.io.git
    branch: master
